# Анализ клиентского опыта (Customer Experience) и обратной связи Т-Банка

Репозиторий содержит материалы аналитического проекта по исследованию удовлетворенности клиентов (CSAT), локализации системных инфраструктурных сбоев («точек боли») в продуктовых вертикалях банка и визуализации геораспределения пользователей. На основе результатов исследования построен интерактивный BI-дашборд.

## Задача и цели проекта

**Задача исследования:** Провести комплексный многофакторный анализ деперсонализированных логов обратной связи из социальных сетей и внешних финансовых платформ для оптимизации процессов клиентского сервиса, снижения раннего оттока пользователей и повышения лояльности к бренду.

**Основные цели проекта:**
1. **Продуктовый аудит:** Локализовать наиболее критичные технические и сервисные дефекты в разрезе бизнес-линий и конкретных продуктов банка.
2. **Сегментация аудитории:** Исследовать поведение и уровень удовлетворенности различных групп клиентов (новые пользователи, премиум-сегмент, возрастные когорты).
3. **Репутационный анализ (SERM):** Сравнить тональность и структуру клиентских обращений на внешних площадках разного типа.
4. **Автоматизация ETL:** Разработать отказоустойчивый скрипт предобработки сырых данных для их корректного геокодирования и импорта в BI-систему.

##  Ключевые вопросы, на которые отвечает исследование

В ходе работы над проектом были даны ответы на следующие бизнес-вопросы:
* **География:** Какова реальная плотность трансграничной активности клиентов банка и где фиксируются локальные кластеры обращений в странах СНГ?
* **Каналы коммуникации:** Чем принципиально отличается поведение клиентов на профессиональных финансовых порталах (Banki.ru, Sravni.ru) от их активности в социальных сетях (ВКонтакте, Одноклассники)?
* **Ценность сервиса:** Насколько критично обстоят дела с качеством обслуживания в высокодоходном Premium-сегменте банка?
* **Инфраструктура:** Какие конкретные технические подтемы (ошибки авторизации, проблемы с СБП, скрытые комиссии) генерируют наибольший объем негатива?

## Источники данных

Для реализации проекта использовался деперсонализированный массив данных в формате `.xlsx`, содержащий операционные метрики и текстовые логи. 

### Основные анализируемые поля датасета:
* `id_client` — зашифрованный идентификатор пользователя.
* `review_source`, `review_text`, `review_emotion`, `review_mark` — параметры текстовой обратной связи и оценки.
* `csat_score` — расчетный индекс удовлетворенности.
* `company`, `business_line`, `product`, `reason`, `subtheme` — продуктовая иерархия и классификатор инцидентов.
* `solution_flg` — маркер успешного закрытия жалобы.
* `new_flg`, `subscription_important_flg` — маркеры сегментации (новый клиент, премиум).
* `gender_cd`, `age_segment` — социально-демографические атрибуты.
* `citizenship_country` — исходное зашумленное поле географии (обработано через Python до `citizenship_country_fixed`).

## Технологический стек
* **Python (Pandas)** — автоматизированный ETL-процесс: очистка данных, нормализация регистров текстовых полей, жесткий маппинг некорректных геотопонимов и аббревиатур (РФ $\rightarrow$ Россия, РБ $\rightarrow$ Беларусь и т.д.) для предотвращения ошибок `Null/NaN` при импорте.
* **Yandex DataLens** — проектирование реляционной модели данных, расчет вычисляемых показателей, агрегация метрик и построение дашборда с интерактивной фильтрацией и картографией.
